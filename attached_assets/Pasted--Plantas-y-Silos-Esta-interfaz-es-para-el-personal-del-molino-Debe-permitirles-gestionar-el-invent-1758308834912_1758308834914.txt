"Plantas y Silos"
Esta interfaz es para el personal del molino. Debe permitirles gestionar el inventario físico y digital del arroz, manteniendo una trazabilidad perfecta de cada lote.

1.1. Layout y Flujo Principal
La pestaña se organiza en dos paneles verticales:

Panel Izquierdo: Una lista seleccionable de "Plantas Industriales" (Molinos) a las que el usuario tiene acceso. Incluye un botón + Agregar Planta.

Panel Derecho: Muestra los detalles y silos de la planta seleccionada en el panel izquierdo. Si no hay ninguna seleccionada, muestra el mensaje "Seleccione una planta para gestionar sus silos".

1.2. Panel Derecho: Gestión de la Planta Seleccionada
Al seleccionar una planta, este panel se activa y muestra:

Título: El nombre de la planta seleccionada (ej: "Planta Industrial SAMAN").

Menú de Acciones Globales:

Botón + Agregar Silo: Abre un modal para definir un nuevo silo.

Botón Registrar Proceso Industrial: Abre un modal para registrar operaciones que afectan a todo un silo (ej: Secado, Aireación, Limpieza), registrando fecha y parámetros.

Selector de Lógica de Trasiego: Un componente Radio.Group o Tabs que define cómo se calcula la composición del arroz al moverlo. Esta selección afecta a toda la planta y es crucial para la trazabilidad.

Opción 1: Mezcla Proporcional: Al extraer un porcentaje del volumen de un silo, el sistema asume que se extrae ese mismo porcentaje de cada lote individual que contiene.

Opción 2: Manejo por Capas (FIFO): El sistema calcula la extracción basándose en el orden de entrada. Los lotes más antiguos (los de abajo) salen primero. Este cálculo requiere el diámetro del silo para estimar las capas.

Visualización de Silos: Un grid con un componente SiloCard por cada silo de la planta.

1.3. Componente: SiloCard
Cada card es una representación visual y funcional de un silo.

Header: Muestra el ID del Silo y su Capacidad Máxima.

Contenido Principal:

Un indicador visual del nivel de llenado (ej: Progress bar) con el texto "Ocupación: 4,500 / 10,000 Toneladas".

Una Tabla de Lotes, que es el núcleo de esta vista. Cada fila representa un camión (remito) descargado en el silo.

Columnas: ID Lote (ID del remito), Chacra Origen, Variedad, Tonelaje (t), Fecha Entrada.

Footer (Acciones del Silo):

Botón Realizar Trasiego / Asignar a Venta: Abre el modal de extracción.

1.4. Modales y Lógica de Interacción
Modal "Agregar/Editar Silo":

Campos de entrada: ID Silo (texto), Tipo de Silo (desplegable), Capacidad Máxima (t) (numérico), y Diámetro (m) (numérico). El campo Diámetro es obligatorio, ya que es indispensable para la lógica de "Manejo por Capas".

Modal "Realizar Trasiego / Asignar a Venta":

Campos de entrada:

Cantidad a Extraer (t) (numérico).

Destino (menú desplegable): Debe mostrar una lista de todos los demás silos en la misma planta y una opción llamada "Asignar a Orden de Venta".

Cálculo de Composición (Vista Previa): Antes de confirmar, el modal debe mostrar una tabla de solo lectura detallando exactamente qué lotes y qué cantidad de cada uno se moverán. Este cálculo debe ejecutarse automáticamente basado en la Lógica de Trasiego (Mezcla o Capas) seleccionada a nivel de planta. El usuario no puede editar este desglose, solo confirmarlo.

Confirmación: Al confirmar, el sistema actualiza el estado de los silos (origen y destino, si aplica) y mantiene un registro inmutable de la transacción para auditoría.
